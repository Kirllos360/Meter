# P0-001 — Wire TOU Pricing to Billing Engine

## Goal
Connect the Time-of-Use pricing structure (already in TariffCharge/TariffChargeDetail tables) to the billing engine so invoices calculate TOU rates instead of flat rates.

## Scope
- Read TOU periods from TariffChargeDetail
- Match reading timestamp to TOU period
- Calculate consumption per period
- Apply period-specific rates
- Generate invoice line items per period
- Does NOT include: demand charges, penalties, credit notes

## Business Rules
- TOU periods: Peak, Off-Peak, Shoulder (configurable per tariff)
- Reading timestamp determines which period applies
- Consumption is split across periods proportionally
- Each period appears as separate invoice line item
- Fall back to flat rate if no TOU periods configured

## Architecture Rules
- No cross-area data access
- All mutations audited
- Backward compatible — existing flat-rate tariffs unaffected

## Database
- Tables: TariffCharge (period_type), TariffChargeDetail (period_start, period_end, rate)
- No new tables needed
- Index: TariffCharge(tariff_id, period_type)

## Backend
- Files: `billing/tariff-calculation.service.ts`, `billing/calculation-engine.service.ts`
- New method: `calculateTOU(consumption, readings, tariffId) → InvoiceLine[]`
- Refactor: `calculateInvoice()` to call TOU when tariff has periods

## Frontend
- No changes needed (TOU lines appear as invoice line items)
